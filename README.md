# 教学协作指南

## 如何使用这个系统

### 1. 规划新课程

告诉我：
- 课程名称和目标学生
- 总课时和每周时长
- 核心知识点

我会帮你：
- 生成完整教学大纲
- 设计每周 slides 结构
- 准备代码示例和练习题

### 2. 制作单节课件

告诉我：
- 课程主题
- 这节课的具体目标
- 需要多少 slides（建议 8-15 张）

我会生成：
- 完整的 slides 大纲
- 每页内容（文字 + 代码）
- 讲师备注和时间分配

### 3. 迭代优化

你可以：
- "这页太复杂，简化一下"
- "加一个实际案例"
- "给这个算法配个图"
- "生成 3 道练习题"

### 4. 导出使用

目前支持：
- **Markdown 格式**（通用）
- **纯文本大纲**（粘贴到任何工具）
- **HTML 格式**（浏览器展示）

你可以复制到：
- Google Slides
- PowerPoint
- Keynote
- Canva

## 示例对话

**你：** 我要给 Monash FITxxxx 做第 3 周的课件，主题是递归算法，90 分钟。

**我：** 好的，这是 12 张 slides 的结构...

**你：** 第 5 页的代码例子太简单了，换个实际的。

**我：** 已更新，用文件系统遍历作为例子...

**你：** 再加 2 道课堂练习题。

**我：** 这是练习题和参考答案...

## 文件命名规范

- 教学大纲：`syllabus.md`
- 课件：`week03_recursion_slides.md`
- 练习：`week03_exercises.md`
- 答案：`week03_solutions.md`
