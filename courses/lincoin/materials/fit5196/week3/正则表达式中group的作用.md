在正则表达式（regex）里，**group（分组）**的作用可以总结为三个主要方面：

## **1.** **结构化匹配\*\***（把一段模式当作一个整体）\*\*

- 用 () 把一段模式包起来后，这段模式就能被当作一个整体处理。

- 好处是：

  - 可以对整个组应用量词（\*、+、?、{m,n}）。
  - 避免量词只作用在前一个字符上。

**例子**

```
(ab)+
```

匹配："ab"、"abab"、"ababab"

如果没有括号，ab+ 匹配 "a" + "b+"（即 "ab"、"abb"、"abbb"），含义就完全不同。

---

## **2.** **捕获匹配内容\*\***（之后可以复用）\*\*

- 默认括号是 **捕获分组**，会记录它匹配到的内容，方便之后：

  - 在替换中引用（$1、\1）
  - 在代码里提取（如 Python match.group(1)）
  - 在正则内部引用（反向引用 backreference）

**例子**

```
(\d{4})-(\d{2})-(\d{2})
```

匹配 "2025-08-10"，捕获到：

1. 2025（年）
2. 08（月）
3. 10（日）

替换例子（把日期变成 10/08/2025）：

```
$3/$2/$1
```

---

## **3.** **控制优先级\*\***（改变默认匹配逻辑）\*\*

- 正则里的 |（或）运算符优先级很低，常需要用 () 限制范围。

- 没有括号时，a|bc 匹配 "a" 或 "bc"；

  有括号 (a|b)c 则匹配 "ac" 或 "bc"。

**例子**

```
gr(a|e)y
```

匹配 "gray" 或 "grey"

如果写成 gra|ey，含义就跑偏了。
